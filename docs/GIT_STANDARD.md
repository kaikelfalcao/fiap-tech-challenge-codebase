# Git Standards

To keep the repository organized, follow these branch and commit standards:

## Branches

- **main** – production-ready code  
- **feature/** – new features or improvements (e.g., `feature/user-crud`)  
- **fix/** – bug fixes or hotfixes (e.g., `fix/user-exception`)

## Commit Messages

Use clear and consistent commit messages following a conventional style. Recommended types:

- **feat** – new feature  
- **fix** – bug fix  
- **docs** – documentation  
- **style** – formatting  
- **refactor** – code change without feature or fix  
- **test** – tests  
- **chore** – maintenance  

For more details and examples of good commit messages, see:  
[Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)
